1. Overview:

Insufficient Credential Hygiene refers to poor practices around password and authentication management that increase the risk of account compromise. This includes weak passwords, password reuse, default credentials, and improper storage of credentials.

2. Vulnerability from an Attacker Perspective:

Attackers exploit weak credential hygiene to gain unauthorized access: Weak Passwords: Easily guessable (123456, password), allowing brute-force attacks. Password Reuse: Compromise of one account leads to compromise of other accounts using the same password. Default Passwords: Vendor or factory credentials left unchanged can be leveraged for easy access. Insecure Password Storage: Passwords stored in plaintext or weak hashes can be stolen in a breach, enabling credential reuse.

2.1 Attacker Attempts:

- Brute-force login attempts using common passwords. - Credential stuffing using leaked passwords from other services. - Searching for default credentials in devices, applications, or IoT hardware. - Exploiting weak hash algorithms (MD5, SHA1) to crack passwords from a database dump.

3. Vulnerability from a Defender Perspective:

- Lack of strong password policies. - Absence of multi-factor authentication (MFA). - Failure to hash and salt passwords before storing in the database. - Not disabling or changing default passwords on systems or applications.

3.1 Solutions for a Defender:

- Enforce strong, unique passwords for every account (e.g., minimum length, complexity requirements). - Encourage or require use of password managers for secure password generation and storage. - Hash passwords securely using slow, modern algorithms (bcrypt, Argon2, scrypt) with unique salts. - Disable or immediately change all default credentials on devices, software, and services. - Implement multi-factor authentication (MFA) wherever possible. - Monitor for reused credentials or compromised passwords via threat intelligence feeds.

4. Extras:

- Weak credential hygiene is a leading cause of account compromise in data breaches. - Even a single reused or weak password can lead to cascading compromises. - Training and awareness for users about proper password hygiene is essential. - Combine with session management and access control best practices to strengthen overall security posture.
