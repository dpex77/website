Command Injection

Occurs mainly due to improper Input Validation: Checking TLFR type, format, range, or pattern.

Command injection occurs when an attacker is able to execute arbitrary system commands on a server or host by exploiting un sanitized input in an application.

Application passes user input directly to a system shell or command interpreter.

If input is not properly validated or sanitized, attackers can append commands.

Examples

import os

user_input = "google.com; rm -rf /"
os.system("ping -c 4 " + user_input)

Attacker provides: google.com; rm -rf /
The program runs as: ping -c 4 google.com; rm -rf /

import os

user_command = input("Enter the command: ")  
print("Running your command...\n")  
os.system(user_command)

user types "ifconfig", it gets it. great but attacker. passing unchecked input directly into a shell command.

import os

user_command = input("Enter the command: ")  
os.system("echo " + user_input)

and user input is: ; rm -f *

In a shell (like Bash or CMD), the operator && means: "Run the command on the left, and if it succeeds, run the one on the right."

Dangerous Shell Characters

  • ; Run multiple commands sequentially
  • `` (backticks) Execute command inside them first
  • $(...) Same as backticks — runs inner command

Preventions from Defensive Perspective

  1. Never trust user input
    Treat all input as hostile until proven otherwise.
    Validate against strict allow-lists, not block-lists.
  2. Input validation
    Ensure input matches expected patterns/types.
    Example: if input should be a filename → only allow alphanumeric, dots, underscores.
    Reject shell metacharacters (; | > < $ ( ) { } [ ] * ?).
  3. Input sanitization
    Escape or neutralize unsafe characters before passing to a shell.
    Example: safely escape arguments with libraries (shlex.quote in Python, parameterized APIs in other languages).
  4. Avoid dynamic command execution
    Do not build shell commands with string concatenation (os.system("cmd " + user_input)).
    Use safe APIs that do not invoke a shell:
    Python → subprocess.run(["ls", "-l", filename])
    Java → ProcessBuilder 
import subprocess

cmd = input("Enter safe command like 'ipconfig': ") 
try:
    result = subprocess.run([cmd], check=True)
except Exception as e:
    print("Error:", e)

This still trusts user input — but doesn't invoke a full shell, so it's safer than os.system().

  1. Least privilege
    Run the process with minimal permissions (non-root, chroot/containerized).
    Prevent lateral movement if exploited.
  2. Contextual encoding (if shell execution is unavoidable)
    Properly encode input before passing it into the shell so it’s treated as data, not code.
    Example: quotes around filenames → "safe filename.txt"
  3. Monitoring & detection
    Look for suspicious process execution (e.g., ;, &&, unusual commands).
    Alert on unexpected child processes spawned by web apps (like sh, bash, powershell).
