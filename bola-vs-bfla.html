1. Overview:

Access control vulnerabilities can be divided into object-level and function-level issues: - BOLA: Broken Object Level Access – unauthorized access to other users’ objects/resources. - BFLA: Broken Function Level Access – unauthorized access to restricted functions/features. - BOPLA: Broken Object & Privilege Level Access – a combination where attackers exploit both object-level and privilege-level access simultaneously for maximum impact.

2. Vulnerability from an Attacker Perspective:

BOLA: Access another user’s object by manipulating IDs, filenames, or tokens. Example: GET /api/user/1234/profile # User A's profile GET /api/user/5678/profile # User A now sees User B's profile BFLA: Access a restricted function or feature without proper role check. Example: POST /api/admin/deleteUser # Normal user tries to delete another user GET /upload # Only admins should access BOPLA: Exploit both object and privilege weaknesses. For instance, a low-privilege user manipulates object IDs while invoking admin-only functionality. Example: POST /api/admin/updateUser?user_id=5678 # User A modifies User B's data even though they are not admin Impact: Complete unauthorized control, data manipulation, and potential full compromise of sensitive resources.

2.1 Attacker Attempts:

BOLA: - Enumerate object identifiers (IDs, UUIDs). - Modify parameters to access other users’ data. BFLA: - Directly call restricted endpoints. - Exploit weak role checks. BOPLA: - Combine object manipulation and function abuse. - Chain attacks: e.g., exploit IDOR while executing admin-only actions. - Target both resource-level and privilege-level weaknesses simultaneously.

3. Vulnerability from a Defender Perspective:

BOLA: Missing or weak object-level authorization. BFLA: Function-level access not properly enforced. BOPLA: Applications failing to enforce both object-level ownership and privilege-based function restrictions. This allows attackers to bypass multiple layers of protection.

3.1 Solutions for a Defender:

BOLA Mitigations: - Implement strict server-side object authorization. - Use opaque, unpredictable identifiers. - Validate ownership on each request. BFLA Mitigations: - Enforce RBAC on all sensitive functions. - Validate roles server-side. - Principle of least privilege. BOPLA Mitigations: - Combine BOLA and BFLA protections. - Ensure object-level access checks and function-level authorization are performed together. - Test endpoints for chained attacks. - Regular audits and logging of all privileged actions and sensitive object access.

4. Extras:

- BOLA Question: “Can I access someone else’s object?” → Resource-level issue. - BFLA Question: “Can I perform a function I shouldn’t?” → Action/feature-level issue. - BOPLA Question: “Can I manipulate another user’s object while performing restricted actions?” → Combined object & privilege issue. - Part of OWASP Broken Access Control category. - Often used together with injection or session attacks for maximum impact.
