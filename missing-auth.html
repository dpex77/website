1. Overview:

Missing Authentication for Critical Functions occurs when important actions in an application or system can be executed without verifying the identity of the user. This exposes high-impact functionality to anyone, including unauthenticated attackers, potentially compromising security, integrity, or availability.

2. Vulnerability from an Attacker Perspective:

From an attacker’s perspective, missing authentication provides direct access to critical operations without restriction. Examples of Exploitable Actions: - System upgrade or software patch deployment - Configuration changes or server settings - Account deletion or creation - System shutdown or restart - Modifying sensitive data (financial, personal, or operational) High-value Impact: - Full system compromise - Unauthorized data modification or deletion - Service disruption or downtime - Bypassing security controls meant for privileged users - Escalation opportunities when combined with other vulnerabilities

2.1 Attempts of an Attacker:

- Discover unprotected endpoints (GET, POST, DELETE) using fuzzing or browsing API documentation. - Attempt direct requests to critical functions without providing authentication tokens, sessions, or credentials. - Use tools like Burp Suite or curl to test access to endpoints that should be restricted. - Chain with other vulnerabilities (e.g., privilege escalation, injection) to maximize impact. Example: # Vulnerable endpoint: POST /upgrade # No authentication check in place # Attacker can trigger upgrade directly: curl -X POST http://target.com/upgrade

3. Vulnerability from a Defender Perspective:

Defenders must ensure that all critical functions are protected and never accessible without proper authentication. Common Mistakes: - Assuming network-level controls are sufficient to restrict access. - Exposing APIs to internal or external users without token/session validation. - Lack of role-based access control (RBAC). - Insufficient logging of sensitive operations. - Relying solely on client-side validation (JavaScript checks).

3.1 Solutions for a Defender:

1. Enforce Authentication: - Require login, API tokens, or session verification before executing critical functions. 2. Role-Based Access Control (RBAC): - Assign roles and permissions to restrict which users can perform sensitive actions. 3. Validate Access Tokens: - Check validity of sessions, JWTs, or API keys on every request. 4. Logging and Auditing: - Record all access attempts to critical functions, successful or failed, for monitoring and incident response. 5. Security Testing: - Include missing authentication tests in your security assessment and penetration testing. - Use automated scanners to detect unprotected endpoints.

4. Extras:

- Even internal endpoints should enforce authentication; internal networks are not inherently trustworthy. - Combine with strong authorization checks to prevent privilege abuse. - Always assume an attacker can reach endpoints and enforce authentication and authorization consistently. - OWASP Top 10: This vulnerability falls under “Broken Access Control” (A01:2021).
